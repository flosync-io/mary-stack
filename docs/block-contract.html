Two rules make it a system, not three projects

RULE 1

Pin the version

Mary's behaviour is engine(mary.yml) × knowledge.json — a cross-product. Every finding cites a run. No manifest, no finding. "Mary failed A7" is anecdote; v5.1 × DMC80FD-01@2026-06-23.a failed A7 can be re-run and proven closed.

RULE 2

Route by fault class

Your own maxim — every wrong move is data, logic, or prompt — is the router that sends a finding to the block that owns the fix:

dataknowledge.json wrong / missing→ Block 2
logicengine moved wrong on good data→ Block 1 · engine
promptInterpret mis-typed / Render mis-phrased→ Block 1 · seams
