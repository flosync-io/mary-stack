Current architecture (v5.1, as-is)

Runtime pipeline — one pass per operator message
Start → N0 Read knowledge (HTTP) → N0b Pick → N1 Interpret (LLM) → N2 GuardPre
      → N6 MatchScore (LLM) → N8 Engine → N12 Render (LLM) → N14 Envelope
      → Answer → N13 Persist → N15 Write runtime (HTTP)

Deterministic Interpret·Match·Diagnose. LLM only interprets (in) and renders (out); code decides every move. Constants: DECLINE_FLOOR 0.45, TIE_BAND 0.10. 16/16 engine tests pass.

Why it needs reworking

Split-brain fix target

Thread list reads Supabase (get_threads), but message history live-reads Dify (dify-proxy/messages). Two sources.

Durability hole data loss seen

Deleting old Dify apps permanently lost their transcripts; only structured summaries survived.

Two persistence models drift

May three-table and June two-JSON designs coexist in everyone's head.

Capture depended on resolution null fields

vault diagnostic fields sat NULL because they only wrote if the loop completed; machine_memory had duplicate writes.

dify-proxy (working as designed)

Allowlisted pass-through for 3 routes (chat-messages, messages, files/upload), verify_jwt + requireUser, blocking mode, DIFY_API_KEY server-side. App 1d5aed45-…. The 401s were auth, not a bug.

The promotion loop (B-zero)
Resolve a conversation → flips status=resolved (runtime.json is already full).
promote.py reads resolved runtime → routes by engine state: sep≥0.45 & chosen → ENRICH (→REVISE if contradicts); chosen=null → ADD.
review.html generated — changelog + before→after + inline edit.
Denver reviews (skims adds/enriches, scrutinizes ⚠ revises), edits, approves.
knowledge.json downloaded with _meta version bump + parent_version; commit to version, git revert to roll back.
Versioning & rollback

Never overwrite in place. _meta.version + parent_version chain; commit = version, git revert = rollback. Substrate survives: re-run promote.py to rebuild from runtime.json.

Multi-machine

machine_id is the partition key everywhere. Engine/schema/prompts/floors shared; knowledge/runtime/registry per machine. Onboard N = registry row + knowledge/<N>.json, no code change.

Decisions & deferred

ADR-0001 — knowledge persistence (Proposed)
  • machine_memory dropped; curated knowledge lives only in knowledge/<machine>.json.
  • vault → thin per-conversation index; body in runtime.json.
  • Tiers: runtime (raw) → cache (staged, deferred) → knowledge (curated).
  • Promotion routes on the engine's existing match; one writer to knowledge.json; distillation generalizes, never appends.
  • Rollback: superseded_by → versioned builds + pointer → delta log → re-distill. B-zero = git.
