The promotion loop

Flow
runtime.jsonstatus=green → promote.py→ review.htmlDenver → knowledge.json+v, git

runtime.json is already full (written every turn). Resolve just flips status=green → eligible. promote.py reads the engine's typed state — no LLM.

Routing — from the engine's own match
  • matched_fm set + agrees → ENRICH (+provenance, +field note)
  • …contradicts / escalated → REVISE ⚠ (flagged, starts unchecked)
  • matched_fm null → ADD (new failure mode)
  • non-green → skipped
The review — human-readable decided

Each change shows the session as a plain narrative (operator's own words, slugs resolved), the before→after object (editable), and a checkbox. Denver skims adds/enriches, scrutinizes ⚠ revises, edits inline, approves.

Versioning & rollback

Approve → download knowledge.json with _meta version bump + parent_version + approved_by, plus a decision log. Commit = version. git revert = rollback. Substrate survives: re-run promote.py to rebuild from runtime. One writer to knowledge.json.

Granularity tiers
runtime.jsonraw / every turn cache.jsonstaged · deferred knowledge.jsoncurated · versioned

B-zero skips cache: resolved runtime distills straight to a review. Distillation generalizes — it never appends raw cases into the contract.

get-machine repoint do before dropping tables

It reads machines + fat vault + machine_memory. Keep its JSON shape; source session detail from runtime.json; memory_captured ← runtime.json.resolution_note; drop the machine_memory join.

Tables
  • machine_memory → dropped (curated → knowledge.json)
  • vault → thin index (powers get_threads)
  • machines/operators → registries exist → extend
  • operator_memory → orphan, kept
Identity + errors

Extend machines with dify_app_key, knowledge_path, onboarded. Proxy resolves machine/operator before forwarding. Error envelope: 404 machine_not_onboarded · 404/422 operator_not_found · 401 unauthenticated · 502 upstream_error — no more silent-empty.

runtime.json — the substrate give it a schema

It's engine state, not the transcript (no turns, no operator_id, no timestamps, id is null). Good for distillation, not for history display. Add contracts/runtime.schema.json: required-now id, operator_id, timestamps; nice-to-have ruled_out, match score, knowledge_version.
